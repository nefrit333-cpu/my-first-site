# Политика безопасности

Этот проект является учебным frontend-проектом и представляет собой статический сайт-портфолио.

Несмотря на учебный формат, в проекте важно соблюдать базовые правила безопасности.

## Что считается проблемой безопасности

К проблемам безопасности относятся:

- случайно добавленные пароли
- случайно добавленные токены
- случайно добавленные API-ключи
- случайно добавленные файлы `.env`
- публикация приватных данных
- публикация личных данных без необходимости
- подозрительные зависимости
- вредоносный код
- небезопасные внешние ссылки

## Что нельзя добавлять в репозиторий

В репозиторий нельзя добавлять:

- пароли
- токены
- приватные ключи
- API-ключи
- файлы `.env`
- личные документы
- временные архивы с приватными данными
- папку `node_modules`
- файлы с доступами к сторонним сервисам

## Защита от случайной публикации секретов

В проекте уже используется `.gitignore`.

Он помогает не добавлять в репозиторий:

- `.env`
- временные файлы
- системные файлы
- папку `node_modules`
- архивы
- логи

Перед каждым коммитом нужно проверять статус Git:

```bash
git status
```
