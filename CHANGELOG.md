Дата обновления: 2026-07-07

### Добавлено

- создана первая GitHub Issue для улучшения сайта
- создана отдельная рабочая ветка `feature/improve-projects-block`

- добавлен отдельный файл `projects.css`
- обновлён блок проектов в `index.html`
- добавлены карточки статистики перед проектами
- добавлены улучшенные визуальные карточки проектов
- добавлены статусы проектов с индикаторами
- добавлены дополнительные метки проектов
- добавлены блоки результата для каждого проекта
- добавлено более подробное описание первого опубликованного проекта
- добавлена связь Pull Request с Issue через `Closes #1`

### Улучшено

- блок проектов стал визуально заметнее
- первая карточка проекта стала выглядеть как главный кейс
- будущие проекты стали понятнее по статусу и назначению

- улучшена адаптивность блока проектов
- проект начал использовать рабочий процесс через Issue и Pull Request

### Техническая база

В проект добавлен отдельный CSS-файл для улучшений блока проектов:

```text
projects.css
```

В `index.html` подключён новый файл стилей:

```html
<link rel="stylesheet" href="projects.css" />
```

Работа выполнена через отдельную ветку:

```text
feature/improve-projects-block
```

Pull Request:

```text
#2 — Улучшил блок проектов
```

Issue:

```text
#1 — [Feature]: Улучшить блок проектов
```

### Что изменилось в блоке проектов

- вводный блок с кратким описанием

- улучшенные hover-состояния


### Что дают шаблоны Issues

Шаблон ошибки помогает описать:

- что работает неправильно
- как повторить ошибку
- какое поведение ожидалось
- что происходит фактически
- устройство и браузер
- дополнительные детали

Шаблон улучшения помогает описать:

- идею
- пользу изменения
- примерное поведение
- раздел сайта или файл
- дополнительные материалы

### Что даёт шаблон Pull Request

Шаблон Pull Request помогает описать:

- что изменено
- зачем это нужно
- как проверить изменения
- какие проверки качества выполнены
- какие Issues связаны с изменением
- нужны ли скриншоты

- репозиторий стал ближе к реальному frontend-проекту
- появилась первая автоматическая проверка качества проекта

### Техническая база

В проекте теперь есть настройки редактора:

```text
.editorconfig
```

Есть настройки Prettier:

```text
.prettierrc
.prettierignore
```

Есть npm-конфигурация:

```text
package.json
package-lock.json
```

Есть GitHub Actions workflow:

```text
.github/
└── workflows/
    └── format-check.yml
```

### Команды проекта

Проверить форматирование:

```bash
npm run format:check
```

Автоматически отформатировать файлы:

```bash
npm run format
```
