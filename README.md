Это мой первый учебный сайт-портфолио, который развивается через реальные frontend-задачи.

Проект опубликован через GitHub Pages и постепенно дорабатывается как полноценная витрина навыков начинающего frontend-разработчика.

- блок “Обо мне”

- модальные окна проектов
- контактная форма

- клиентская валидация формы
- защита формы через honeypot
- кнопка возврата наверх
- видимые focus-состояния
- SEO meta-теги
- Open Graph
- Twitter Card
- favicon
- robots.txt
- sitemap.xml
- публикация через GitHub Pages

### Учебный лендинг услуги

Создан отдельный коммерческий учебный лендинг:

```text
Создание сайта для малого бизнеса
```

Лендинг содержит:

- первый экран с оффером
- CTA-кнопки
- технологические теги
- блок “Для кого этот лендинг”
- блок преимуществ
- блок “Что входит”
- этапы работы
- блок доверия
- FAQ
- форму заявки
- полноценный footer
- ссылку на портфолио
- мобильную навигацию
- кнопку возврата наверх

Отдельные файлы лендинга:

```text
landing.html
landing.css
landing.js
```

### Форма лендинга

В версии `v2.9` форма заявки на лендинге стала полноценной frontend-задачей.

Реализовано:

- отдельный JavaScript-файл `landing.js`
- клиентская проверка имени
- проверка email
- проверка длины сообщения
- ошибки под конкретными полями
- очистка ошибки при вводе
- общий статус формы
- перевод фокуса в первое поле с ошибкой
- управление `aria-invalid`
- связь полей и ошибок через `aria-describedby`
- состояние кнопки `Отправляем...`
- отправка данных через `fetch`
- сбор данных через `FormData`
- заголовок `Accept: application/json`
- обработка успешной отправки
- обработка ошибки отправки
- очистка формы только после успешной отправки
- сохранение данных при ошибке
- скрытое honeypot-поле
- служебные поля `_subject`, `source` и `page`

### Мобильный интерфейс лендинга

В версии `v2.9` мобильная версия лендинга была комплексно переработана.

Улучшено:

- мобильная шапка
- круглая кнопка перехода в портфолио
- бургер-меню

- равномерные отступы справа и слева
- отдельное визуальное оформление каждой секции
- разные фоны блоков “Что входит” и “Доверие”
- стабильная сетка карточек
- отображение карточек в одну колонку
- мобильные кнопки первого экрана
- технологические теги
- форма заявки
- footer
- прозрачная кнопка возврата наверх

Проверялись ширины:

```text

```

### Исправления мобильной версии

Исправлены проблемы:

- меню больше не блокирует прокрутку страницы
- меню закрывается при движении пальцем
- кнопка возврата наверх отображается на телефоне
- карточки больше не исчезают
- карточки больше не сжимаются в узкую полосу
- между карточками нет чрезмерных пустых зон
- элементы не выходят за границы секций
- блоки имеют одинаковые поля справа и слева
- footer не обрезается
- hover-эффекты не вызывают смещение карточек на сенсорных экранах

### Визуальный стиль лендинга

Лендинг оформлен как отдельный frontend-кейс.

Используются:

- мягкие `radial-gradient` фоны
- отдельные цветовые схемы секций
- стеклянные элементы
- полупрозрачные кнопки
- аккуратные границы
- мягкие тени
- скруглённые карточки
- hover-эффекты на desktop
- active-эффекты на мобильных устройствах
- одинаковый визуальный ритм
- почти прозрачная кнопка возврата наверх

### SEO лендинга

В версии `v2.8` была улучшена техническая SEO-база лендинга:

- улучшен `title`
- улучшен `meta description`
- расширены `keywords`
- проверен canonical URL
- добавлены Open Graph meta-теги
- добавлены Twitter Card meta-теги
- добавлен social preview
- добавлен alt-текст preview-изображения
- страница `landing.html` добавлена в `sitemap.xml`
- обновлён `lastmod`
- добавлена отдельная запись sitemap для лендинга

- работа с формами
- проверка кода
- постепенное развитие проекта

Проекты оформлены как frontend-кейсы.

Для проектов показываются:

### FAQ

FAQ реализован в формате раскрывающегося аккордеона.

Поддерживаются:

- открытие по клику

- изменение иконки
- плавное раскрытие ответа
- aria-атрибуты

### Контакты

Основная контактная форма умеет:

- проверять имя
- проверять email
- проверять сообщение
- показывать ошибки под полями
- подсвечивать ошибочные поля
- очищать ошибку при вводе
- переводить фокус в первое поле с ошибкой
- показывать состояние отправки
- отправлять заявку через Formspree
- показывать успешное сообщение
- сохранять данные при ошибке
- защищаться от простых ботов через honeypot

## Доступность интерфейса

В проекте реализованы:

- видимые focus-состояния
- навигация через `Tab`
- обратная навигация через `Shift + Tab`
- закрытие модального окна через `Escape`
- удержание фокуса внутри модального окна
- возврат фокуса после закрытия окна
- `aria-expanded`
- `aria-controls`
- `aria-hidden`
- `aria-current`
- `aria-describedby`
- `aria-invalid`
- `aria-live`
- поддержка `prefers-reduced-motion`

Проверка выполнялась через:

```text
Tab
Shift + Tab
Enter
Escape
```
