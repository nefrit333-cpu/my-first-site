## Основные разделы сайта

### Главный экран

Первый экран представляет меня как frontend-разработчика в обучении и показывает текущий фокус: HTML, CSS, JavaScript, Git и GitHub Pages.

В версии `v1.7` главный экран стал компактнее на мобильных устройствах.

### Обо мне

Раздел рассказывает о моём пути в веб-разработке, целях обучения и текущем фокусе.

На мобильной версии улучшены отступы, размер текста и общая читаемость блока.

### Навыки

В блоке навыков показано, какие технологии я изучаю и как применяю их в проекте.

В версии `v1.7` карточки навыков стали аккуратнее на телефонах.

- улучшенная адаптивность

В версии `v1.7` дополнительно улучшена мобильная версия карточек проектов.

В версии `v1.8` улучшена доступность кнопок “Подробнее” и модального окна проектов.

## Рабочий процесс разработки

Проект уже прошёл три полноценных рабочих цикла через GitHub:

```text
Issue → feature-ветка → изменения → Pull Request → merge в main
```

### Цикл 1 — улучшение блока проектов

Для улучшения блока проектов была создана задача:

```text
Issue #1 — [Feature]: Улучшить блок проектов
```

Для разработки была создана отдельная ветка:

```text
feature/improve-projects-block
```

Изменения были отправлены через Pull Request:

```text
Pull Request #2 — Улучшил блок проектов
```

После проверки Pull Request был объединён в основную ветку:

```text
main
```

### Цикл 2 — улучшение мобильной версии

Для улучшения мобильной версии была создана задача:

```text
Issue #3 — [Feature]: Улучшить мобильную версию сайта
```

Для разработки была создана отдельная ветка:

```text
feature/improve-mobile-version
```

Изменения были отправлены через Pull Request:

```text
Pull Request #4 — Улучшил мобильную версию сайта
```

После проверки Pull Request был объединён в основную ветку:

```text
main
```

### Цикл 3 — улучшение доступности интерфейса

Для улучшения доступности интерфейса была создана задача:

```text
Issue #5 — [Feature]: Улучшить доступность интерфейса
```

Для разработки была создана отдельная ветка:

```text
feature/improve-accessibility
```

Изменения были отправлены через Pull Request:

```text
Pull Request #6 — Улучшил доступность интерфейса
```

После проверки Pull Request был объединён в основную ветку:

```text
main
```

Это приближает проект к реальному рабочему процессу frontend-разработки.
