## v3.0 — Тарифы, калькулятор стоимости и передача расчёта в заявку

Дата обновления: 2026-07-13

### Добавлено

#### Тарифы

- отдельная секция тарифов
- тариф `Старт` стоимостью от 25 000 ₽
- тариф `Бизнес` стоимостью от 45 000 ₽
- тариф `Премиум` стоимостью от 75 000 ₽
- примерные цены и описания
- списки включённых работ
- кнопки выбора тарифов
- метка `Популярный`
- выделение рекомендуемого тарифа `Бизнес`
- пункт `Тарифы` в header
- пункт `Тарифы` в footer
- отдельный файл `pricing.css`
- data-атрибуты для синхронизации тарифов

#### Калькулятор стоимости

- отдельная секция калькулятора
- выбор базового тарифа через radio-кнопки
- шесть дополнительных услуг через checkbox
- автоматический пересчёт итоговой стоимости
- форматирование суммы через `Intl.NumberFormat`
- итоговый список выбранных параметров
- кнопка сброса расчёта
- возврат тарифа `Бизнес` после сброса
- кнопка переноса расчёта в заявку
- доступный статус через `aria-live`
- пункт `Калькулятор` в header
- пункт `Калькулятор` в footer
- отдельный файл `calculator.css`

#### Связь с формой заявки

- синхронизация тарифных карточек с калькулятором
- блок выбранной конфигурации рядом с формой
- отображение выбранного тарифа
- отображение дополнительных услуг
- отображение предварительной стоимости
- кнопка очистки конфигурации
- скрытое поле `selected_plan`
- скрытое поле `selected_extras`
- скрытое поле `estimated_price`
- передача расчёта через `FormData`
- отправка данных расчёта через Formspree
- доступное сообщение об изменении конфигурации

### Улучшено

#### Пользовательский сценарий

- путь пользователя теперь проходит от тарифов к калькулятору и форме
- кнопка тарифа выбирает соответствующий тариф
- калькулятор синхронизируется с выбранной карточкой
- старые дополнительные услуги очищаются при выборе тарифа из карточки
- изменения калькулятора обновляют уже перенесённую конфигурацию
- при отсутствии услуг показывается текст `Без дополнительных услуг`
- стоимость отображается с разделителями тысяч

#### Форма заявки

- выбранный расчёт входит в отправляемый `FormData`
- имя, email и сообщение сохраняются при очистке конфигурации
- конфигурация сохраняется при ошибке отправки
- конфигурация сбрасывается после успешной отправки
- скрытые поля очищаются вместе с конфигурацией
- пользователь видит, какие данные будут отправлены

#### Доступность

- для тарифов используются доступные ссылки-кнопки
- для выбора тарифа используются radio-кнопки
- для дополнительных услуг используются checkbox
- применены `fieldset` и `legend`
- итог калькулятора обновляется через `aria-live`
- конфигурация формы обновляется через `aria-live`
- кнопка очистки работает с клавиатуры
- добавлены заметные `focus-visible` состояния
- важная информация не передаётся только цветом
- поддерживается `prefers-reduced-motion`

#### Адаптивность

- тарифы отображаются в три колонки на desktop
- тарифы переходят в одну колонку на mobile
- калькулятор использует двухколоночную структуру на desktop
- итоговый блок калькулятора закрепляется справа
- на планшете и телефоне блок результата переходит вниз
- настройки калькулятора выстраиваются в одну колонку
- блок конфигурации формы адаптирован для узких экранов
- длинные названия услуг переносятся
- стоимость не выходит за границы
- предотвращена горизонтальная прокрутка

### Исправлено

- внутренняя точка выбранной radio-кнопки выровнена по центру
- точка остаётся по центру на desktop и mobile
- кнопка калькулятора теперь действительно переносит расчёт в форму
- тарифные карточки больше не оставляют старые дополнительные услуги
- очищение конфигурации не удаляет имя, email и сообщение
- при ошибке отправки расчёт не теряется

## v2.9 — Форма, FAQ, доступность и производительность лендинга

Дата обновления: 2026-07-11

### Добавлено

- отдельный JavaScript-файл `landing.js`
- клиентская валидация формы лендинга
- ошибки под полями имени, email и сообщения
- общий статус формы
- состояние кнопки `Отправляем...`
- отправка формы через `fetch`
- сбор данных через `FormData`
- обработка успешной и неуспешной отправки
- honeypot-поле
- служебные поля `_subject`, `source` и `page`
- мобильное бургер-меню лендинга
- круглая кнопка перехода в портфолио
- кнопка возврата наверх
- новый footer лендинга
- интерактивный FAQ-аккордеон
- четвёртый вопрос FAQ
- премиальные метки разделов
- skip-link к основному содержимому
- управление закрытой мобильной навигацией через `inert`
- fallback для Tab-навигации через `tabindex`
- автозакрытие FAQ портфолио при клике вне блока

### Улучшено

#### Форма

- добавлена проверка имени
- добавлена проверка email
- добавлена проверка длины сообщения
- ошибки показываются под конкретными полями
- ошибка очищается при вводе
- первое поле с ошибкой получает фокус
- обновляется `aria-invalid`
- поля связаны с ошибками через `aria-describedby`
- форма очищается только после успешной отправки
- при ошибке введённые данные сохраняются

#### Мобильный интерфейс

- улучшена мобильная шапка
- меню закрывается при прокрутке и движении пальца
- бургер закрепляется справа сверху
- страница прокручивается при открытом меню
- выровнены поля секций
- карточки стабильно выстраиваются в одну колонку
- исправлена мобильная сетка
- улучшены CTA-кнопки
- улучшены технологические теги
- улучшены фоны секций
- обновлён footer
- кнопка наверх стала почти прозрачной

#### FAQ и метки

- статичные FAQ-карточки заменены аккордеоном
- при открытии нового вопроса предыдущий закрывается
- повторное нажатие закрывает ответ
- клик вне FAQ закрывает ответ
- `Escape` закрывает ответ
- добавлены `aria-expanded`, `aria-controls` и `aria-hidden`
- добавлены `aria-labelledby` и `role="region"`
- без JavaScript ответы остаются доступными
- добавлена поддержка `prefers-reduced-motion`
- метки получили полупрозрачный фон, мягкие границы и тени
- добавлен лёгкий блик на desktop
- добавлены mobile active-состояния
- яркая заливка меток в тёмных секциях убрана

#### Доступность

- добавлена skip-link
- основное содержимое получило отдельный идентификатор
- улучшены `focus-visible` состояния
- при открытии меню фокус переходит на первый пункт
- после закрытия через `Escape` фокус возвращается на бургер
- закрытая навигация исключается из Tab-навигации
- обновляются `aria-expanded` и `aria-hidden`
- сохранена доступность без JavaScript

#### Производительность

- обновления scroll и resize ограничены через `requestAnimationFrame`
- исключено создание повторных кадров
- кэшируются состояния шапки и кнопки наверх
- DOM обновляется только при смене состояния
- оптимизирован resize открытого FAQ
- тяжёлые hover-эффекты ограничены устройствами с мышью
- transform-эффекты отключены на сенсорных устройствах
- декоративные анимации отключаются через `prefers-reduced-motion`

### Исправлено

- блокировка прокрутки мобильным меню
- отсутствие кнопки наверх на телефоне
- смещение секций
- неравномерные поля
- пересечение мобильных блоков
- исчезновение карточек
- сжатие карточек в узкие полосы
- большие пустые промежутки
- нестабильная мобильная отрисовка
- обрезание footer
- зависшие hover-состояния на сенсорных экранах
- яркая заливка меток “Преимущества” и “Заявка”
- FAQ портфолио, который оставался открытым после клика по другому разделу
