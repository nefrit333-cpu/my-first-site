## v3.2 — Пошаговый бриф, сохранение черновика и расширенная заявка

Дата обновления: 2026-07-18

### Добавлено

#### Пошаговый бриф проекта

- отдельная секция `Бриф проекта`
- пять последовательных шагов
- выбор типа сайта
- выбор основной задачи
- выбор нескольких возможностей
- выбор желаемого срока
- выбор ориентировочного бюджета
- radio-кнопки и checkbox
- кнопки `Назад` и `Продолжить`
- проверка обязательных ответов
- сообщения об ошибках через `aria-live`
- прогресс `Шаг N из 5`
- визуальная полоса прогресса
- итоговая сводка
- кнопка изменения ответов
- кнопка полной очистки
- кнопка добавления брифа в заявку
- пункт `Бриф` в header
- пункт `Бриф` в footer
- отдельный файл `brief.css`

#### Автоматический черновик

- ключ `dmitriy-web-project-brief-v1`
- версия структуры черновика
- дата последнего сохранения
- сохранение текущего шага
- сохранение состояния итоговой сводки
- автоматическое сохранение radio
- автоматическое сохранение checkbox
- восстановление после перезагрузки
- срок хранения 30 дней

- статус последнего сохранения
- форматирование времени через `Intl.DateTimeFormat`
- сообщение об изменении в другой вкладке через событие `storage`
- fallback при недоступном `localStorage`

#### Расширенная заявка

- карточка `Бриф добавлен в заявку`
- отображение пяти ответов возле формы
- кнопка `Изменить бриф`

- скрытое поле `brief_status`

- отправка структурированных полей через Formspree
- текстовая сводка между служебными маркерами

#### Hover-система портфолио

- hover для полей формы
- согласованные эффекты карточек
- эффекты внутренних блоков проектов
- hover для тегов, FAQ, кнопок и ссылок
- hover переключателя темы и кнопки меню
- hover кнопки модального окна и кнопки наверх
- отдельные active-состояния для touch-устройств
- ограничение декоративных эффектов через `@media (hover: hover) and (pointer: fine)`

### Улучшено

#### Пользовательский сценарий

- пользователь проходит от заполнения брифа к структурированной заявке
- одновременно отображается только один шаг
- после смены шага фокус переносится на его заголовок
- сводку можно проверить до добавления в форму
- повторное добавление обновляет старые данные
- собственный текст сообщения сохраняется
- карточка брифа не дублируется
- текстовая сводка не дублируется

#### Сохранение состояния

- пустой бриф не сохраняется
- недопустимые значения не восстанавливаются
- устаревший черновик удаляется автоматически
- удаление сохранённой копии не очищает ответы на экране
- полная очистка удаляет ответы и локальную запись
- восстановление не перехватывает фокус
- имя, email и сообщение не записываются в `localStorage`
- тариф и выбранный кейс не записываются в `localStorage`

#### Форма заявки

- тариф, выбранный проект и бриф работают независимо
- удаление брифа из заявки не очищает исходные ответы
- удаление брифа не очищает локальный черновик
- удаление брифа не очищает тариф и выбранный проект
- очистка тарифа не удаляет бриф
- очистка выбранного проекта не удаляет бриф
- после успешной отправки очищается только контекст формы
- после успешной отправки исходный бриф сохраняется
- после успешной отправки локальный черновик сохраняется
- при ошибке отправки весь контекст остаётся на экране

#### Доступность

- шаги используют `fieldset` и `legend`
- прогресс использует `role="progressbar"`
- скрытые шаги используют `aria-hidden` и `inert`
- ошибки, переходы и изменения сообщаются через `aria-live`
- управление работает через `ArrowLeft`, `ArrowRight`, `Home`, `End` и `Escape`
- восстановление черновика не забирает фокус
- скрытые поля не участвуют в Tab-навигации
- hover не заменяет `focus-visible`
- touch-устройства не зависят от hover
- поддерживается `prefers-reduced-motion`

#### Адаптивность

- варианты брифа переходят в одну колонку
- кнопки занимают доступную ширину
- итоговая сводка адаптируется под узкие экраны
- карточка брифа возле формы переходит в одну колонку
- длинный список возможностей переносится
- эффекты карточек не создают горизонтальную прокрутку

### Исправлено

- повторное добавление брифа больше не создаёт дубликат текста
- повторное добавление не создаёт вторую карточку
- удаление контекста не удаляет собственный текст пользователя
- повреждённый черновик не ломает интерфейс
- неизвестная версия данных не восстанавливается
- старый черновик не хранится бесконечно
- недоступный `localStorage` не блокирует заполнение
- очистка одного контекста заявки не удаляет остальные
- успешная отправка не удаляет исходный бриф
- hover-эффекты не зависают на touch-устройствах
